\hypertarget{pip_named_export}{}\subsubsection{pip\-\_\-named\-\_\-export}\label{pip_named_export}
export an address of the calling Pi\-P root or a Pi\-P task to the others.

\begin{DoxyParagraph}{Synopsis}
\#include $<$\hyperlink{pip_8h_source}{pip.\-h}$>$ \par
int pip\-\_\-named\-\_\-export( void $\ast$exp, const char $\ast$format, ... )
\end{DoxyParagraph}
\begin{DoxyParagraph}{Description}
Pass an address of a memory region to the other Pi\-P task. Unlike the simmple {\ttfamily pip\-\_\-export} and {\ttfamily pip\-\_\-import} functions which can only export one address per task, {\ttfamily pip\-\_\-named\-\_\-export} and {\ttfamily pip\-\_\-named\-\_\-import} can associate a name with an address so that Pi\-P root or Pi\-P task can exchange arbitrary number of addressess.
\end{DoxyParagraph}

\begin{DoxyParams}[1]{Parameters}
\mbox{\tt in}  & {\em exp} & an address to be passed to the other Pi\-P task \\
\hline
\mbox{\tt in}  & {\em format} & a {\ttfamily printf} format to give the exported address a name. If this is {\ttfamily N\-U\-L\-L}, then the name is assumed to be \char`\"{}\char`\"{}.\\
\hline
\end{DoxyParams}
\begin{DoxyReturn}{Returns}
Return 0 on success. Return an error code on error. 
\end{DoxyReturn}

\begin{DoxyRetVals}{Return values}
{\em E\-P\-E\-R\-M} & {\ttfamily pip\-\_\-init} is not yet called. \\
\hline
{\em E\-B\-U\-S\-Y} & The name is already registered. \\
\hline
{\em E\-N\-O\-M\-E\-M} & Not enough memory\\
\hline
\end{DoxyRetVals}
\begin{DoxyNote}{Note}
The addresses exported by {\ttfamily pip\-\_\-named\-\_\-export} cannot be imported by calling {\ttfamily pip\-\_\-import}, and vice versa.
\end{DoxyNote}
